## DM Sequences

### Warm DM Sequence (After Public Engagement — PREFERRED)

**Step 1: Public reply (value-first, no pitch)**

Give genuine advice. Never mention Suprdash in the public reply.

Example on a "my website isn't converting" post:
```
Three quick things I'd check:
1. Is your CTA above the fold? Most visitors don't scroll.
2. Page load time — over 3 seconds and you lose 40% of visitors.
3. Hero copy needs to answer "what's in it for me" in under 5 seconds.
Run PageSpeed Insights — surfaces the biggest issues fast.
```

**Step 2: DM (same day, 1-4 hours after public reply)**

```
Hey [name] — saw your post about [specific issue]. Didn't want to spam the thread, but I took a quick look at [their site if shared].

A couple things jumped out:
- [Specific observation about their site]
- [Another specific observation]

I co-founded a web dev shop (suprdash.com) — we work with startups and SMBs on everything from quick Framer builds to custom React/Next.js apps. Happy to hop on a quick call and give you an honest assessment of what you actually need. No pitch, just a second set of eyes.

Interested? Here's my calendar: [link]
```

**Step 3: Follow-up (3-5 days later, ONLY if seen but no reply)**

```
Hey [name] — just following up. Totally understand if timing isn't right. We've built sites for startups like FieldKonnect and Fenmo AI — happy to share what we learned if it's ever useful.

No pressure at all.
```

**Step 4: Final touch (7-10 days after step 3, ONLY if previous read)**

```
Last one from me — if the website situation comes back up, our calendar's at suprdash.com. Good luck with [their project]!
```

## Reply Templates (Public — Value-First)

### "My website sucks" / "Website doesn't convert"

```
A few things that usually make the biggest difference:

1. [Specific technical advice for their situation]
2. [Tool or resource recommendation]
3. [Common mistake they're likely making]

Happy to take a closer look if you want a second opinion.
```

### "Should I use Framer/Webflow/custom?"

```
Honest answer: depends on what you need.

If it's a marketing site, landing page, or portfolio — Framer/Webflow gets you there faster and cheaper. No question.

Where it breaks: custom features, integrations, user auth, anything beyond a marketing site. That's when you need actual development.

What are you building? I can give you a more specific recommendation.
```

### "How much does a website cost?"

```
Varies wildly depending on scope. A landing page and a custom web app are completely different projects.

Best way to get a real answer: get 3 quotes from agencies/devs after sharing your actual requirements. Be specific about features, pages, and integrations.

What are you building? I can help you figure out what category your project falls into.
```

**NOTE: Never share specific pricing numbers. Drive toward a call.**

### "Looking for a developer"

```
A few questions that'll help you find the right fit:

1. What kind of site? (marketing/landing page vs web app vs e-commerce)
2. Do you have designs already or need design + dev?
3. Timeline — when do you need it live?

These determine whether you need a freelancer, a small agency, or a dev shop. Happy to help narrow it down.
```

### "Just launched — feedback?"

```
First off — shipping is the hardest part, so congrats.

A few things I'd tweak:
- [Specific observation about their site]
- [Another observation]
- [Performance/mobile note if relevant]

The core product looks [honest assessment]. These are mostly quick wins that would make a real difference.
```

## Disqualification (Polite Decline)

### Budget Mismatch

```
Appreciate the interest! Based on what you're describing, a DIY builder like Framer or Squarespace might be the best fit for where you are right now. They've gotten really good.

If you outgrow it later, we're around. Good luck with the build!
```

### Not a Fit (Student/Learning)

```
Nice project! For learning purposes, I'd recommend [specific tool/tutorial]. When you're ready for production work, feel free to reach out.
```
