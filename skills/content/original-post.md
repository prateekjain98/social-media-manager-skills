# Original Post Drafting

Draft original posts for X accounts based on persona, LinkedIn cross-posting, and content pillars.

## Skill: `/original-post`

**CRITICAL: Read the persona's `_persona.md` first. All content must match the persona's voice, stories, and content pillars. Never sound AI-generated.**

---

## Human-Authenticity Rules

### Sound Like a Real Founder
- Keep posts short and crisp (2-5 sentences max)
- No em dashes, no AI buzzwords (leverage, unlock, game-changer)
- Use specific numbers always ($13K MRR, 250K installs, $50K/year)
- Write like talking to a co-founder, not writing an essay
- Each sentence should hit hard on its own line
- Use line breaks between sentences for visual spacing on X

### Formatting for X
- One thought per line
- Blank line between each sentence/thought for readability
- Strong opening line (hook) — no "I" starts if possible
- Closing line should be punchy, memorable, quotable
- Stay under 280 characters per tweet (or use thread format for longer)
- Always capitalize the first letter of every sentence

### AVOID
- Starting with "I think" or "In my opinion"
- Generic motivational fluff
- Overuse of hashtags (0-1 max)
- Em dashes (—)
- Corporate speak or LinkedIn-style professional tone
- Mentioning product in every post (1-in-5 rule)

---

## When Invoked

### 1. Gather Context
- Read the active persona file (`_persona.md`)
- Check content pillars and story bank
- Review LinkedIn posts for cross-posting ideas (if available)
- Check what's trending on X in the persona's niche
- Review recent posts to avoid repetition

### 2. Draft Options
Generate **3-4 post options** across different content pillars:

**Option types to rotate:**
- **Contrarian take** — challenge a popular belief with evidence
- **Story snippet** — real experience, specific numbers, lesson learned
- **Observation** — pattern you've noticed in the industry
- **Builder update** — what you're working on (without being spammy)
- **Cross-post from LinkedIn** — adapt LinkedIn content for X's shorter format

### 3. Present to User
Show all options clearly labeled (Option A, B, C, D) with the content pillar noted.

### 4. Post to X
- Type the chosen option into X compose window with proper line spacing
- Each sentence on its own line with blank lines between
- Wait for user's "Hit post" approval before clicking Post button

## Quality Checklist
- [ ] Would a real founder post this? (not too polished)
- [ ] Does it have specific numbers or details?
- [ ] Is each line punchy enough to stand alone?
- [ ] No AI-sounding language?
- [ ] Follows 1-in-5 product mention rule?
- [ ] Proper line spacing for X readability?
- [ ] Under 280 chars (or clearly a thread)?
